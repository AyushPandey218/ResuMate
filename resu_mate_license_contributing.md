### 4Ô∏è‚É£ Development Guidelines

Please follow these guidelines when contributing:

- Keep the app **100% client-side** (no backend additions without discussion)
- Do not introduce tracking, analytics, or user data collection
- Reuse existing UI components where possible
- Follow existing code style and folder structure
- Ensure changes do not break PDF export or preview rendering

---

### 5Ô∏è‚É£ Commit Message Style

Use clear and descriptive commit messages:

```text
feat: add print-mode spacing improvements
fix: prevent PDF layout overflow on long content
refactor: simplify resume context state
```

---

### 6Ô∏è‚É£ Test Before Submitting

Before opening a pull request, make sure:

- App runs without errors (`npm run dev`)
- Resume preview renders correctly
- PDF export works and is not broken
- No console errors or warnings

---

### 7Ô∏è‚É£ Open a Pull Request

Push your branch and open a PR against the `main` branch:

- Clearly describe **what** you changed
- Explain **why** the change is needed
- Add screenshots or screen recordings if UI-related
